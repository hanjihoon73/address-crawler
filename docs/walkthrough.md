# 개발 완료 및 사용 가이드 (Walkthrough)

## 프로젝트 개요
네이버 지도의 특정 검색어에 대한 장소 정보를 수집하여 리스트로 확인하고 엑셀로 저장할 수 있는 웹 애플리케이션입니다.

## 구현된 기능
1. **지도 데이터 크롤링**
   - Puppeteer를 활용하여 네이버 지도 검색 결과 자동 수집
   - "V" 버튼(상세보기) 자동 펼침 기능을 통한 **지번 주소** 우선 추출
   - 도로명 주소, 상호명, 카테고리 정보 수집

2. **UI/UX**
   - 검색어 및 수집 개수(10~200개) 설정 기능
   - 실시간 진행 상태 표시 (로딩 인디케이터)
   - 수집된 데이터 테이블 뷰 및 페이지네이션 (20개 단위)
   - **엑셀 다운로드** 기능 (`.xlsx`)

3. **기술 스택**
   - Frontend: Next.js 14 (App Router), CSS Modules
   - Backend: Next.js API Routes, Puppeteer
   - Utils: xlsx (Excel Export), lucide-react (Icons)

## 실행 방법

### 1. 의존성 설치 (최초 1회)
터미널에서 아래 명령어를 실행하여 필요한 패키지를 설치합니다.
(이미 설치가 진행 중이라면 건너뛰세요.)

```powershell
npm install
```

### 2. 개발 서버 실행
설치가 완료되면 아래 명령어로 개발 서버를 실행하세요.

```powershell
npm run dev
```

### 3. 웹사이트 접속
브라우저를 열고 다음 주소로 접속합니다.
http://localhost:3000

## 사용 시 주의사항
- **크롤링 속도**: 브라우저를 실제로 띄워서 데이터를 수집하므로, 수집 개수에 따라 시간이 소요될 수 있습니다. (20개 기준 약 10~20초)
- **브라우저 창**: 수집 중 크롬 브라우저가 실행됩니다. 수집이 완료될 때까지 브라우저를 닫지 마세요.
- **네이버 UI 변경**: 네이버 지도의 UI 구조가 변경될 경우 동작하지 않을 수 있으며, 이 경우 `src/utils/crawler.ts`의 선택자(Selector) 수정이 필요합니다.

## 폴더 구조
```
src/
├── app/
│   ├── api/crawl/    # 크롤링 API 엔드포인트
│   ├── page.tsx      # 메인 페이지
│   └── globals.css   # 전역 스타일
├── components/
│   ├── search/       # 검색 폼 컴포넌트
│   ├── result/       # 결과 테이블 컴포넌트
│   └── common/       # 엑셀 버튼 등 공통 컴포넌트
├── hooks/            # 커스텀 훅 (useMapCrawler)
└── utils/            # 크롤러 로직 (crawler.ts)
```
